# Governança do Projeto

## Objetivo

Este documento estabelece os padrões utilizados no projeto QA Testes Manuais.

Seu objetivo é garantir organização, padronização e rastreabilidade dos artefatos produzidos durante a execução das atividades de Qualidade de Software.

---

## Escopo

Esta governança aplica-se a todos os documentos, casos de teste, execuções, evidências, relatórios de bugs, matrizes e demais artefatos presentes neste repositório.

---

## Estrutura do Projeto

O projeto está organizado nas seguintes áreas:

- Casos de Teste
- Plano de Testes
- Execução de Testes
- Evidências
- Relatórios de Bugs
- Documentação
- Matrizes
- Planejamento
- Templates
- Sprints

---

## Convenção de Nomes

Todos os artefatos seguem um padrão de identificação.

Exemplos:

- CT-001
- ET-001
- BUG-001
- MT-001

Os nomes devem ser claros, objetivos e representar corretamente o conteúdo do arquivo.

---

## Controle de Versão

Todo o versionamento é realizado através do Git e GitHub.

Os commits devem possuir mensagens descritivas que representem claramente a alteração realizada.

Exemplo:

- docs: adiciona governança do projeto
- test: adiciona execução do ET-003
- bug: registra BUG-002

---

## Organização das Evidências

Todas as evidências devem:

- possuir nomenclatura padronizada;
- estar vinculadas à execução de teste correspondente;
- preservar informações relevantes para validação;
- ocultar dados sensíveis quando necessário.

---

## Registro de Bugs

Todo bug registrado deve conter:

- identificação;
- descrição;
- passos para reprodução;
- resultado esperado;
- resultado obtido;
- evidências;
- severidade;
- prioridade.

---

## Atualização

Este documento poderá ser atualizado conforme a evolução do projeto e adoção de novas práticas de Qualidade de Software.
