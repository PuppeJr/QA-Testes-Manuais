# Critérios de Entrada e Saída

## Objetivo

Este documento define os critérios mínimos para iniciar e finalizar a execução dos testes do projeto QA-Testes-Manuais.

---

## Critérios de Entrada

Os testes somente poderão ser iniciados quando todos os itens abaixo forem atendidos.

| ID     | Critério                  | Status |
| ------ | ------------------------- | ------ |
| EC-001 | Requisitos documentados   | ☐      |
| EC-002 | Ambiente disponível       | ☐      |
| EC-003 | Sistema acessível         | ☐      |
| EC-004 | Casos de Teste revisados  | ☐      |
| EC-005 | Massa de Testes preparada | ☐      |
| EC-006 | Plano de Testes aprovado  | ☐      |

---

## Critérios de Saída

A execução será considerada concluída quando todos os itens abaixo forem atendidos.

| ID     | Critério                             | Status |
| ------ | ------------------------------------ | ------ |
| XC-001 | Todos os testes críticos executados  | ☐      |
| XC-002 | Evidências registradas               | ☐      |
| XC-003 | Bugs documentados                    | ☐      |
| XC-004 | Matriz de Rastreabilidade atualizada | ☐      |
| XC-005 | Relatório Final atualizado           | ☐      |

---

## Observações

Caso algum critério de entrada não seja atendido, a execução dos testes deverá ser interrompida até sua regularização.

Da mesma forma, os testes somente serão encerrados após o atendimento de todos os critérios de saída.
